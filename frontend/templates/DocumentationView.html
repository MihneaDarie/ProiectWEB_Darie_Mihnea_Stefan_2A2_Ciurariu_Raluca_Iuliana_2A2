Abstract

This project represents a comprehensive web platform for generating, visualizing and managing datasets of various types including numeric arrays, character arrays, matrices, graphs and trees. The system integrates a modern JavaScript frontend with a PHP backend and Oracle database, providing authentication, data operation and administration functionalities.

1. Introduction

The Problem Input Generator addresses the need for a centralized system to create and manage structured datasets. The platform emphasizes secure data processing and visualization, ensuring data integrity and preventing misinterpretation.

1.1 Project Objectives

  • Provide a user-friendly interface for dataset generation and management
  • Implement secure authentication and role-based access control
  • Ensure reliable data processing and visualization
  • Maintain data integrity through robust validation
  • Offer extensibility for future data types

4. Semantics Overlay

The "Semantics Overlay" represents our approach to secure data processing and visualization, preventing misinterpretation or incorrect display of data.

Key Implementation

The formatParsedOutputSafe(type, rawData) function in frontend/js/profile.js is central to this approach:

  • Normalizes and protects data during display (HTML escaping, type validation)
  • Implements JSON parsing with fallbacks for invalid data
  • Provides clear error messages for corrupted or missing data
  • Maintains clear separation between data types

4.1 Backend Processing

Backend components (GeneratorModel.php and SQL procedures in Pig_db.sql) ensure:

  • Consistent data storage
  • CLOB/JSON handling
  • Format validation during read/write operations

4.2 Importance

  • Prevents code injection or interface corruption
  • Provides users with clear, secure feedback
  • Facilitates easy extension with new data types
